The Panl Response Object

The Panl response is always of the type JSON (with a MIME type of "application/json" with UTF-8 encoding.

On every response there will always be a key of error, with a boolean status of true, or false.  If there is an error (either 404, or 500), then the error key will be set to true, an integer status code keyed on status, and keyed on message, a human readable response.

There will __ALWAYS__ be an error key in all responses which makes it easy to determine the best course of action.

Integration and Implementation

Implementation is straightforward, only requiring that the remove_uri value is used as the href attribute for the anchor tag - i.e.

<a href="/Clutch/page-2/msb+po+/">5-per-page</a>

Notes: To render the text of the anchor tag, the encoded value will need to be URI decoded. In the panl-viewer.js file, additional processing is done thusly:

return(decodeURI(text).replaceAll("+", " ").replaceAll("%2B", "+"));

The above line will replace the + character with spaces (which is not done with the javascript function decodeURI()) then any encoded + characters (i.e. %2B) are replaced with the + character.

Looking at the response:

If you wanted to remove all sorting:

  • remove_uri is the URI path to reset to no sorting - which is the default Solr sort of relevance descending (Line 2)

If you wanted to replace the sorting - i.e. remove the name sorting, and replace it with brand sorting:

  • set_uri_asc is the URI path for ascending (Line 7)
  • set_uri_desc is the URI path for descending (Line 8)

If you wanted to add sorting by brand in addition to the current sort of name - i.e. sort first by name ascending then by brand:

  • add_uri_asc is the URI path for sorting by the current sort order, then by brand ascending (Line 9)
  • add_uri_desc is the URI path for sorting by the current sort order, then by brand descending (Line 10)

Implementation and Integration

RANGE facets have a variety of configuration options, and depending on the requirements of the URI path generation will depend on what properties need to be set.

IMPORTANT: Range facets will __ALWAYS__ be returned in the results.


Without an infix

Depending on the properties that are set,  all of the following URI paths are equivalent and will return exactly the same results. without an infix (this defaults to the ~ character) are equivalent and will return exactly the same results:

  • /10~50/w+w/
  • /10+grams~50+grams/w+w/
  • /weight+10~weight+50/w+w/
  • /weight+10+grams~weight+50+grams/w+w/
  • /from+light~heavy+pencils/w+w/
  • /from+light~50/w+w/
  • /from+light~50+grams/w+w/
  • /from+light~weight+50+grams/w+w/
  • /from+light~weight+50/w+w/
  • /10~heavy+pencils/w+w/
  • /10+grams~heavy+pencils/w+w/
  • /weight+10+grams~heavy+pencils/w+w/
  • /weight+10~heavy+pencils/w+w/

With an infix

Depending on the properties that are set, all of the following URI paths with an infix (set to +to+) are equivalent and will return the exactly the same results:

The order of precedence with an infix set

The order of precedence for the text before the infix is:

  • If it exists, the minimum range value replacement

~ else ~

  • If there is a range prefix, the prefix + the 'from' value

~ else ~

  • If it a value prefix or value suffix exists, the value prefix + value + value suffix

~ otherwise ~

  • Just the value

The order of precedence for the text after the infix is:

  • If it exists, the maximum range value replacement

~ else ~

  • If there is a range suffix, the value  +  the range suffix

~ else ~

  • If it a value prefix or value suffix exists, the value prefix + value + value suffix

~ otherwise ~

  • Just the value

The order of precedence without an infix set

The order of precedence for the text without an infix set is:

  • If it exists, the minimum range value replacement, or
  • If it exists, the value prefix
  • The value
  • If it exists, the value suffix

The order of precedence for the text after the infix is:

  • If it exists, the maximum range value replacement, or
  • If it exists, the value prefix
  • The 'to' value
  • If it exists, the value suffix
